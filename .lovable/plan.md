
# Consent OS — План реализации

Прототип системы управления согласиями на персональные данные. Светлая минималистичная тема, React + TanStack Start, данные в localStorage (без бэкенда).

## Маршруты
- `/` — Дашборд «Мои сервисы» (сетка карточек)
- `/history` — История согласий (timeline)
- `/data-flow` — Визуализация потоков данных (граф «Я → сервисы»)

Общий хедер с навигацией и логотипом «Consent OS».

## Дашборд «Мои сервисы»
Сетка карточек (1 кол. на мобиле, 2–3 на десктопе). Демо-сервисы при первом запуске:
- 🏦 **СберБанк** — паспорт, счета, транзакции — риск 🔴 высокий
- 🏛 **Госуслуги** — ФИО, СНИЛС, адрес — риск 🟡 средний
- 🍔 **Яндекс Еда** — телефон, геолокация, история заказов — риск 🟢 низкий

На каждой карточке:
- Иконка + название сервиса
- Список запрошенных данных (теги-чипы)
- Индикатор риска с цветной точкой и **tooltip** с пояснением (что именно рискованно)
- Статус: Активен / Отозван (badge)
- Дата выдачи согласия
- Кнопка **«Отозвать доступ»** (для отозванных — disabled, карточка приглушена в серый)

Сверху — мини-статистика: всего сервисов / активных / отозванных / уровень риска по портфелю.

## Поток отзыва (one-click revocation)
1. Клик по «Отозвать» → открывается **модалка подтверждения**
2. В модалке: название сервиса + блок «Последствия»: «Сервис больше не сможет видеть ваши данные. Действие можно откатить в течение 30 дней.»
3. Кнопки «Отмена» / «Подтвердить отзыв»
4. При подтверждении:
   - Async-функция симулирует API-вызов: `fetch('https://httpbin.org/post', …)` с payload `{service, action: 'revoke', timestamp}`
   - В консоль: `Webhook отправлен в [сервис]`
   - Toast-уведомление «Доступ отозван»
   - Обновление localStorage: статус → `revoked`, добавляется запись в историю
   - UI перерисовывается, карточка становится серой

Для уже отозванных — кнопка **«Восстановить»** (тоже через модалку, тот же async-флоу).

## Вкладка «История»
Вертикальный timeline с записями:
- Иконка действия (выдан / отозван / восстановлен)
- Сервис, действие, дата+время
- Перечень данных, которые были затронуты
- Фильтры сверху: все / выдачи / отзывы / по сервису

Пустое состояние если истории нет (но при первом запуске сидится 3 записи «согласие выдано»).

## Вкладка «Потоки данных»
Визуализация в чистом CSS/SVG:
- В центре — узел «Вы» (аватар-круг)
- Вокруг — узлы активных сервисов с иконками
- SVG-стрелки от центра к каждому сервису, цвет стрелки = уровень риска
- Подпись на стрелке: какие категории данных передаются
- Отозванные сервисы — пунктир, приглушённые
- Hover на стрелке → tooltip с деталями

## Хранение данных (localStorage)
Два ключа:
- `consent-os:services` — массив сервисов
- `consent-os:history` — массив событий

При первой загрузке (если пусто) — посев демо-данных. Хук `useConsentStore` инкапсулирует CRUD и автоматически синхронизирует с localStorage.

## Дизайн
- Светлая тема, фон `#fafafa`, карточки белые с мягкой тенью
- Акцент — индиго (`#4f46e5`)
- Цвета риска: зелёный/жёлтый/красный
- Шрифт **Inter** (подключение через Google Fonts)
- Скруглённые углы (12–16px), мягкие тени, плавные transitions
- Полностью адаптивно: бургер-меню на мобиле, сетка перестраивается
